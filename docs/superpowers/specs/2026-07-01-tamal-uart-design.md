# Tamal — UART transport (host link) Design

Date: 2026-07-01
Status: Approved (design); implementation not started
Scope: A compile-time-configurable **UART** (RX + TX, 8N1) as the host↔FPGA
transport, in the single `Dom100` (100 MHz) clock domain, using an NCO-generated
16× oversample enable (no PLL, no second clock domain). Four focused modules
under `Tamal.Uart.*`, hedgehog-tested (including a TX→RX loopback). The `topEntity`
UART load/drain FSM, the eSPI Engine, and the `tamal-loader` host side are **out
of scope** (§2, §11).

Companion to the ISA & HDL Engine design
(`docs/superpowers/specs/2026-07-01-tamal-isa-design.md`, esp. §10 — the top's
"UART load/drain FSM") and `AGENTS.md` (Arty A7-100T, USB-UART transport, 100 MHz
`Dom100`, no reset port). Written to be implementable from a fresh session.

---

## 1. Purpose & context

`AGENTS.md` fixes the v1 transport as **USB-UART**: the host loads a compiled
tamal program and control state over the link, and the FPGA streams the result/
trace ring back. The ISA design §10 lists a "UART load/drain FSM" in the impure
`topEntity` shell. This spec builds the **UART itself** — the byte-level serial
link — as a set of pure-cored, independently-testable sequential modules. The
load/drain FSM that sits between the UART and the instruction/ring BRAMs is a
separate, later piece.

The UART is being built **before** the Engine deliberately: it is simpler,
self-contained, and far easier to verify (a byte-exact TX→RX loopback), whereas
the Engine is the stateful difficulty spike.

### 1.1 The governing numbers

- System clock: **100 MHz** (`Dom100`, Arty A7-100T CLK100MHZ / pin E3).
- Default baud: **2 000 000** (2 Mbaud), 8N1.
- `100 MHz / 2 Mbaud = 50` clocks/bit — **exact integer**.
- 16× oversample tick = `16 × 2 Mbaud = 32 MHz`; `100 / 32 = 3.125` — **not**
  an integer. This fractional ratio is why the oversample clock is generated by
  an **NCO enable**, not by integer division and not by a PLL (§3, decision 1).

## 2. Scope & non-goals

**In scope**

- `Tamal.Uart.BaudGen`: `oversampleTick`, the NCO 16×-oversample enable.
- `Tamal.Uart.Rx`: `uartRx`, an 8N1 receiver (start detect, oversample,
  majority-vote sampling, LSB-first deserialize, framing check).
- `Tamal.Uart.Tx`: `uartTx`, an 8N1 transmitter (serialize start/8-data/stop,
  `ready` handshake).
- `Tamal.Uart`: `uart`, the thin top that shares one tick into RX + TX.
- Compile-time **baud** (an `SNat`); 8N1 framing and 16× oversampling fixed.
- Hedgehog tests via Clash simulation, including a TX→RX loopback.
- Cabal / test-runner wiring.

**Out of scope (deferred)**

- The `topEntity` **load/drain FSM** (UART ↔ instruction-BRAM / ring-BRAM) and
  the pin/IOBUF wiring. The UART exposes a clean byte interface; wiring it to
  memory and the drain trigger is the shell's job.
- The eSPI **Engine** and everything downstream.
- **Configurable framing** (data bits, parity, stop bits) and **configurable
  oversampling** — fixed at 8N1 / 16× (YAGNI: the host↔FPGA contract is fixed).
- The **host-side** `tamal-loader` transport backend (a Rust crate, separate).

## 3. Design decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | **Single `Dom100` domain; 16× oversample from an NCO enable, not a PLL.** | `100/32 = 3.125` is fractional, so no integer divider yields 32 MHz. A PLL would create a second clock domain, forcing CDC (2-flop synchronizers + async FIFO) between the UART and the `Dom100` Engine — more hardware, more failure modes, harder to verify — for no benefit. An NCO enable emits exactly 16 ticks per 50-clock bit in one domain with ±1-clock jitter that is irrelevant at 16×. |
| 2 | **RX + TX together**, sharing one oversample tick. | Symmetric, both small, and wiring TX's line back into RX gives a byte-exact loopback property test with no external model. |
| 3 | **Compile-time `baud` only** (`SNat`); 8N1 + 16× fixed; `f_clk` from `DomainPeriod dom`. | The host↔FPGA transport contract is fixed; baud is the one knob you might retune (signal-integrity debugging). Fixing framing keeps the FSMs minimal (no parity, no variable width). Deriving `f_clk` from the domain avoids a redundant, mismatch-prone knob. |
| 4 | **Strobe/handshake interface, RX framing-error strobe.** | At 2 Mbaud a byte spans ~500 `Dom100` cycles, so the consumer always keeps up — full valid/ready backpressure is overkill. RX emits a one-cycle `Just byte`; TX takes a byte + emits `ready`. A cheap framing-error strobe suits a compliance rig (observe a bad transport). |
| 5 | **3-sample majority vote at every bit center** (start-confirm, 8 data, stop), one sampling primitive. | Majority-of-3 (samples at oversample counts 7/8/9) rejects a single glitch/edge at the sample instant — the actual point of 16× oversampling. Reusing it at every bit center avoids special cases. |
| 6 | **RX synchronizes its own input line** (internal 2-flop synchronizer). | The line is asynchronous (from a pin); the synchronizer belongs with the receiver, keeping `uartRx` self-contained and metastability-safe. |
| 7 | **Mealy machines (pure transition + thin `mealy` lift), enabled by the tick.** | Matches the codebase's pure-core philosophy; the transition functions are ordinary total functions, testable via Clash `simulate`. No reset port — power-up `init` sets the idle state (AGENTS.md). |

## 4. Module layout & public interfaces

Four new modules under `hdl/src/Tamal/Uart/` (+ the umbrella `Tamal/Uart.hs`),
each `[sequential]`, carrying the REUSE/SPDX header. `NumericUnderscores` is
enabled per-file (as `Domain.hs` does) for `2_000_000`-style literals.

```haskell
-- SPDX-FileCopyrightText: 2026 Felipe Balbi
-- SPDX-License-Identifier: CERN-OHL-P-2.0
{-# LANGUAGE NumericUnderscores #-}
```

### `Tamal.Uart.BaudGen`

```haskell
-- | 16x-oversample enable: a 1-cycle pulse at (16 * baud) average, generated by
-- an NCO in the system domain. f_clk is taken from the domain period.
oversampleTick
  :: forall baud dom
   . (HiddenClockResetEnable dom, KnownDomain dom, KnownNat baud)
  => SNat baud
  -> Signal dom Bool
```

### `Tamal.Uart.Rx`

```haskell
uartRx
  :: HiddenClockResetEnable dom
  => Signal dom Bool          -- ^ oversample tick (enable)
  -> Signal dom Bit           -- ^ rx line (asynchronous; synchronized internally)
  -> ( Signal dom (Maybe (BitVector 8))   -- ^ received byte, 1-cycle strobe
     , Signal dom Bool )                  -- ^ framing-error strobe (1 cycle)
```

### `Tamal.Uart.Tx`

```haskell
uartTx
  :: HiddenClockResetEnable dom
  => Signal dom Bool                       -- ^ oversample tick (enable)
  -> Signal dom (Maybe (BitVector 8))      -- ^ byte in (accepted when ready && Just)
  -> ( Signal dom Bit                      -- ^ tx line (idles high)
     , Signal dom Bool )                   -- ^ ready (high when idle / can accept)
```

### `Tamal.Uart` (umbrella top)

```haskell
uart
  :: forall baud dom
   . (HiddenClockResetEnable dom, KnownDomain dom, KnownNat baud)
  => SNat baud
  -> Signal dom Bit                        -- ^ rx line in
  -> Signal dom (Maybe (BitVector 8))      -- ^ tx byte in
  -> ( Signal dom (Maybe (BitVector 8))    -- ^ rx byte out (strobe)
     , Signal dom Bool                     -- ^ rx framing error (strobe)
     , Signal dom Bit                      -- ^ tx line out
     , Signal dom Bool )                   -- ^ tx ready
```

`uart` generates the tick once (`oversampleTick baud`) and feeds both `uartRx`
and `uartTx`. The `topEntity` calls `uart (SNat @2_000_000)`.

## 5. `Tamal.Uart.BaudGen` — the NCO

One free-running phase accumulator in `Dom100`, advancing every clock (the NCO
itself is **not** gated — it is what *produces* the enable):

```
inc  = 16 * baud                       -- 32_000_000 at 2 Mbaud
mod  = f_clk                           -- from DomainPeriod dom; 100_000_000 for Dom100
each clock:  s = acc + inc
             if s >= mod  then acc' = s - mod ; tick = True
                          else acc' = s       ; tick = False
```

- Over any 50-clock bit at 2 Mbaud this emits **exactly 16 ticks**
  (`50 × 32/100 = 16`); tick spacing alternates 3/4 clocks (avg 3.125), ±1-clock
  jitter — negligible at 16×.
- `f_clk` is computed from the domain at compile time:
  `f_clk = 1_000_000_000_000 `div` <DomainPeriod dom in ps>` (Dom100 period =
  10 000 ps → 100 MHz). Both `inc` and `mod` are compile-time constants.
- Accumulator sized to hold `mod` (Dom100: `Unsigned 27` suffices; the
  implementation may size it via `CLog 2 (mod + 1)` or use a fixed `Unsigned 32`).
- No reset port; `acc` powers up at 0 via `init`.

## 6. `Tamal.Uart.Rx` — 8N1 receiver

Runs one FSM step **per oversample tick** (its "32 MHz" cadence is the enable).
An internal **2-flop synchronizer** registers the async line before use. A free
sample counter `cnt :: Index 16` advances per tick within a bit window; the bit
center is count 8, and the **sample-and-vote** primitive reads the synchronized
line at counts 7/8/9 and returns the majority.

```haskell
sampleVote s7 s8 s9 = (s7 .&. s8) .|. (s7 .&. s9) .|. (s8 .&. s9)   -- >=2 of 3
```

FSM (`RxState = Idle | Start | Data (Index 8) | Stop`), transitions on ticks:

| State | Behaviour |
|-------|-----------|
| `Idle` | line high; on synchronized line **low**, → `Start`, reset `cnt = 0`. |
| `Start` | at the bit center, majority-sample; still **low** ⇒ real start → `Data 0`, realign `cnt`; **high** ⇒ glitch → `Idle`. |
| `Data i` | at each bit center, majority-sample and shift the bit in **LSB-first**; after bit 7 → `Stop`. |
| `Stop` | at the bit center, majority-sample; **high** ⇒ emit `Just byte`, `framingErr = 0`; **low** ⇒ `framingErr = 1`, **drop** the byte; → `Idle`. |

- Output byte strobe and framing-error strobe are each asserted for exactly the
  one cycle the `Stop` sample completes.
- LSB-first deserialize: shift the sampled bit into the MSB of an 8-bit register
  and shift right, or index by `i`; the result is `bit i = data bit i`.
- Start detection is checked every tick, so it may lag the true edge by <1 tick
  (<1/16 bit) — absorbed by center sampling.

---

## 7. `Tamal.Uart.Tx` — 8N1 transmitter

Line idles **high**. Holds each level for exactly **16 oversample ticks** (one
bit), counted with `cnt :: Index 16`; the bit index advances on the 16th tick.

FSM (`TxState = Idle | Start | Data (Index 8) | Stop`):

| State | Behaviour |
|-------|-----------|
| `Idle` | drive **high**, `ready = 1`; when input is `Just b` (caller presents it while `ready`), latch `b`, `ready = 0`, → `Start`. |
| `Start` | drive **low** for 16 ticks → `Data 0`. |
| `Data i` | drive `b[i]` (**LSB-first**) for 16 ticks; after bit 7 → `Stop`. |
| `Stop` | drive **high** for 16 ticks → `Idle` (`ready = 1`). |

- One-cycle handshake: the caller asserts `Just b` on a cycle when `ready` is
  high; TX latches `b` that cycle. `ready` stays low until the stop bit finishes.
- Byte acceptance is gated on `ready` so a `Just` presented mid-transmission is
  ignored (no corruption).

## 8. `Tamal.Uart` — top wiring

```haskell
uart baud rxLine txByte =
  let tick             = oversampleTick baud
      (rxByte, rxErr)  = uartRx tick rxLine
      (txLine, txRdy)  = uartTx tick txByte
   in (rxByte, rxErr, txLine, txRdy)
```

RX and TX share the single free-running tick. TX bit boundaries land on tick
multiples; RX aligns its center sampling to the start bit it detects — so in
loopback (TX line → RX line) each TX bit (held 16 ticks) is sampled dead-center
by RX.

## 9. Engine / shell integration contract (informative)

Not built here; recorded so the interface is the right shape.

- **RX → load path.** The shell's load FSM consumes `rxByte` strobes to fill the
  instruction BRAM (and control state); `rxErr` can abort/flag a corrupt load.
- **TX ← drain path.** The drain FSM presents ring words as bytes on `txByte`
  while `txReady` is high, little-endian per ISA §7 (ring drain). Backpressure is
  `txReady`: present the next byte only when ready.
- **Domain.** Everything is `Dom100`; no CDC. The Engine (also `Dom100`) and the
  UART share the clock, so bytes cross between them as plain `Dom100` signals.
- **Pins.** The `topEntity` maps `rxLine`/`txLine` to the USB-UART pins and does
  any IO registering; `uartRx` already synchronizes its input, so the shell need
  only wire the pad.

## 11. Testing (hedgehog + Clash simulation)

These are sequential, so tests drive them with Clash simulation
(`sampleN`/`simulate`/`bundle` under `withClockResetEnable systemClockGen
resetGen enableGen`) rather than the pure-function style. New `Test.Uart` module.

- **NCO tick rate** (`oversampleTick`): over `N` cycles the tick count is within
  ±1 of `N·16·baud / f_clk`; and there are exactly 16 ticks in each 50-cycle
  window at 2 Mbaud. (Count `True`s in `sampleN N`.)
- **RX known-frame decode:** build the oversampled 8N1 sample stream for a
  hedgehog byte (start, 8 data LSB-first, stop, each 16 ticks) and assert
  `uartRx` emits `Just byte === input` with `framingErr` low.
- **RX framing error:** same, but drive the stop bit **low** → assert a
  `framingErr` strobe and **no** byte.
- **RX glitch rejection (majority vote):** inject a single flipped sample at one
  bit center → assert the byte still decodes correctly (locks in decision 5).
- **TX waveform:** load a byte, sample `txLine` per bit, assert
  `low, b0..b7 (LSB-first), high`.
- **TX ready handshake:** `ready` high in idle, low during a transmission, high
  again after the stop bit; a `Just` presented mid-transmission is ignored.
- **Loopback (byte-exact, the keystone test):** `uartTx` line → `uartRx` line,
  shared tick; feed a hedgehog byte to TX, run enough cycles, assert RX recovers
  the same byte. Extend to a short list of bytes (back-to-back frames).
